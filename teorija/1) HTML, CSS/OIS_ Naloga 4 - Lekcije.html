Semantične značke HTML5

Prejšnje vaje smo spoznali element <div>, ki se uporablja predvsem za združevanje elementov v logične enote za razporeditev spletnih strani. Dolgoletne izkušnje spletnih dizajnerjev so pokazale, da se določeni odseki spletnih strani ponavljajo v vsaki spletni strani, zato prinaša HTML5 določene nove elemente, ki služijo prav temu namenu. To so:

Odzivno oblikovanje

Število različnih naprav, s katerimi dostopamo do spletnih mest, vsako leto narašča. Če smo pred nekaj leti za brskanje uporabljali zgolj osebne računalnike, je dandanes slika precej drugačna. Ne samo, da za brskanje uporabljamo različne tipe naprav (od katerih so najpogostejši osebni računalniki, prenosniki, tablični računalniki in mobilni telefoni), lahko znotraj posameznega tipa najdemo ogromno različic naprav z različnimi velikostmi in resolucijami zaslonov. Seveda je brskanje odvisno tudi od uporabnika, ki lahko mobilno napravo kadarkoli zasuče in povzroči spremembo v postavitvi formata (pokončno/ležeče).

Kot odgovor na težavo, se je pred nekaj leti pojavil princip odzivnega oblikovanja spletnih strani (ang. Responsive Web Design). Odzivno oblikovanje teži k izdelavi strani z optimalno izkušnjo prikazovanja vsebine. To pomeni enostavno pregledovanje in navigiranje s čim manj premikanja, drsenja in ročnega prilagajanja velikosti strani ne glede na velikost zaslona ali različico naprave.


Rešitve

  • pametno oblikovanje spletnih strani s float, clear, max-width in max-height CSS lastnostmi
  • postavitev po meri glede na velikost
  • prikazovanje in skrivanje vsebin glede na velikost

Prilagajanje CSS glede na velikost zaslona (@media)

S prilagajanjem velikosti elementov lahko naslovimo veliko težav, vendar pa je včasih potrebno spremeniti tudi samo postavitev elementov. To lahko dosežemo z nastavljanjem ločenih stilnih datotek in z uporabo CSS media query.

style.css

 .vsebina { width: 500px; }

 @media screen and (max-width: 600px) {
   .vsebina { width: 100%; }
 }

Preverjamo lahko več možnosti

  • tip naprave (screen, tv, print, handheld ...)
  • najmanjšo širino okna (min-width) in največjo širino okna (max-width)
  • širino naprave (max-device-width, min-device-width), primerno za mobilne naprave
  • orientacijo naprave (orientation: portrait, orientation: landscape)
  • razmerje ekrana (device-aspect-ratio)

Več primerjav med sabo ločimo z besedo and. Pogojev je lahko neomejeno.

Primer

style.css

 .pravokotnik { background: white; }

 /* max-width */
 @media (max-width: 600px) {
   .pravokotnik { background: green; }
 }

 /* min-width */
 @media (min-width: 900px) {
   .pravokotnik { background: red; }
 }

 /* min-width & max-width */
 @media (min-width: 600px) and (max-width: 900px) {
   .pravokotnik { background: blue; }
 }

Namesto mešanja različnih CSS-ov lahko razdelimo CSS lastnosti mišljene za različne velikosti v različne datoteke. V datoteki style.css imamo stile za normalno velikost in v datoteki style_mobile.css imamo stile za mobilne naprave. Vključevanje v spletno stran poteka sledeče.

style.css

<html>
 <head>
   <link rel="stylesheet" href="style.css" />
   <link rel="stylesheet" media="screen and (max-width: 480px)"
      href="style_mobile.css" />
 </head>
 
 ...
 
</html>

Prikazovanje in skrivanje vsebine

Prikazovanje celotne vsebine, ki je na voljo na velikih zaslonih ni nujno najbolj praktično za manjše zaslone. To velja predvsem za mobilne naprave, kjer obstajajo drugačne dobre prakse glede navigacije, fokusiranosti vsebine, seznamov in prikazovanja podatkov. Prav zato prikaz določenih delov vsebine ni najbolj smiseln ali celo potreben na mobilnih napravah. Za doseg tega cilja lahko uporabimo funkcionalnost CSS, ki nam z nastavljanjem atributa »display« omogoča prikazovanje in skrivanje vsebine, glede na zastavljene pogoje. To zapišemo v ločeno CSS datoteko na način iz prejšnjega primera.

style.css

 #content{
   width: 54%;
   float: left;
   margin-right: 3%;
 }

 #sidebar-left{
   width: 20%;
   float: left;
   margin-right: 3%;
 }

 #sidebar-right{
   width: 20%;
   float: left;
 }

 .sidebar-nav{display: none;}

style_mobile.css

 #content{
   width: 100%;
 }

 #sidebar-left{
   display: none;
 }

 #sidebar-right{
   display: none;
 }

 .sidebar-nav{display: inline;}

Mobilne naprave

Nekatere naprave vsilijo "zoom" na spletnih straneh. To nam samo otežuje kreacijo mobilnih verzij, zato s sledečim elementom, znotraj glave, prepovemo brskalniku povečevanje naše spletne strani. Torej če delamo spletno stran, ki bo imela mobilno verzijo, uporabimo to vrstico v glavi.

<head>
  <meta name="viewport" content="width=device-width, initial-scale=1" />
</head>

Mrežna postavitev

Obstaja več načinov kako ustvariti neke vraste mrežno postavitev spletne strani. Najbolj poznani so float, flexbox in grid layout. Primerjavo lahko najdete na sledeči povezavi. V tej lekciji si bomo podrobneje pogledali grid layout.

Mrežna postavitev (angl. grid layout) uvaja dvodimenzionalni mrežni sistem za postavitev elementov. Mreže se lahko uporabljajo za postavitev glavnih logičnih enot strani ali majhnih elementov uporabniškega vmesnika.

Kaj je mreža?

Mreža je presek vodoravnih in navpičnih črt - stolpcev in vrstic. Elemente postavimo na mrežo na točno določenem stolpcu in vrstici – podobno kot pri tabeli. Da je vsebina elementa postavljena v obliko mreže nastavimo s CSS atributom display, ki ga nastavimo na grid. Potem pa s pomočjo CSS atributa grid-template-columns definiramo koliko stolpcev bo mreža vsebovala. Vsaka vrednost tega atributa prikazuje en stolpec - širino tega stolpca. Poglejmo si prvi primer.

<div class="mreza">
  <div>Prvi</div>
  <div>Drugi</div>
  <div>Tretji</div>
  <div>Četrti</div>
  <div>Peti</div>
</div>
.mreza {
  display: grid;
  grid-template-columns: 200px 2fr 1fr;
}

Zgornja koda vrne rezultat, kjer je vsebina div-a mreza razporejen v tri stolpce – prvi stolpec je širok 200 pikslov (200px), drugi stolpec je širok 2 dela preostalega prostora (2fr) in tretji stolpec je širok 1 del preostalega dela (1fr).


Če zgornji spletni strani spreminjamo širino, vidimo, da je prvi stolpec fiksen na 200px, drugi je širok dva dela preostalega prostora, tretji pa je širok en del preostalega prostora.

Črte mreže

Spodnja slika prikazuje črte na mreži. Prva navpična (vertikalna) črta je pred prvim stolpcem, druga navpična črta je med prvim in drugim stolpcem in tako naprej.

 

Zakaj je to pomembno? Ker lahko elemente v mreži raztegnemo od črte do črte – podobno kot pri elementih tabele. Poglejmo si primer.

<div class="mreza">
  <div class="box1">Prvi</div>
  <div class="box2">Drugi</div>
  <div class="box3">Tretji</div>
  <div class="box4">Četrti</div>
  <div class="box5">Peti</div>
</div>
.mreza {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-auto-rows: 100px; /* višina vrstic */ 

  column-gap: 10px;
  row-gap: 20px;
}

CSS atributa column-gap in row-gap pa povesta, koliko razmaka je med elementi- column-gap pove razmak med stolpci, row-gap pa med vrsticami. Z atributom grid-auto-rows pa povemo višino vrstic. Poglejmo si še CSS za box elemente.

.box1 {
  grid-column-start: 1;
  grid-column-end: 4;
  grid-row-start: 1;
  grid-row-end: 3;
}

.box2 {
  grid-column-start: 1;
  grid-row-start: 3;
  grid-row-end: 5;
}

CSS atribut grid-column-start pove elementu, pri kateri navpični črti se začne element. Atribut grid-column-end pove, pri kateri navpični črti se konča element. CSS atributa grid-row-start in grid-row-end prav tako povesta, kje pri vodoravni črti mreže se element začne in konča. Vsi elementi, ki nimajo določenih začetkov in koncev, se preprosto razporedijo tako, da padejo v en del mreže, sledijo pa si od levega zgornjega roba proti spodnjemu desnemu robu.


Zgornji CSS za posamezne elemente mreže je lahko tudi skrajšan, kot kaže spodnja koda.

.box1 {
  grid-column: 1 / 4;
  grid-row: 1 / 3;
}

.box2 {
  grid-column: 1 / 2;
  grid-row: 3 / 5;
}

Prva številka pove začetek črte, nato sledi znak poševnice / in na koncu je črta do katere sega element. Še krajše lahko zapišemo kodo sledeče z CSS atributom grid-area.

.box1 {
  grid-area: 1 / 1/ 3 / 4;
}

.box2 {
  grid-area: 3 / 1 / 5 / 2;
}

Pri grid-area si številke sledijo v sledeč vrstnem redu:

  1. začetek vodoravno,
  2. začetek v navpično,
  3. konec vodoravno,
  4. konec navpično.

Naslavljanje elementov mreže po razredu

Vsakemu elementu znotraj mreže lahko nastavimo naziv s CSS atributom grid-area, kot kaže spodnja koda. Vrednost CSS atributa grid-area je ime elementa, ki ga bomo kasneje pri nastavitvi postavitve naslavljali.

<div class="mreza">
    <div class="glava">Glava</div>
    <div class="meni">Meni z navigacijo</div>
    <div class="vsebina">Vsebina</div>
    <div class="noga">Noga</div>
</div>
.glava {
  background-color: red;
  height: 100px;
  grid-area: hd;
}
.noga {
  background-color: blue;
  grid-area: ft;
}
.vsebina {
  background-color: orange;
  grid-area: main;
}
.meni {
  background-color: green;
  height: 400px;
  grid-area: nav;
}

Nato pa lahko v elementu mreže s pomočjo CSS atributa grid-template-areas »izrišemo« postavitev. Poglejmo si spodnji primer CSS-a za div razreda mreza. Vsaka vrstica zapisa bo svoja vrstica na sami strani.

.mreza {
    display: grid;
    grid-template-columns: 200px 1fr 1fr 1fr 1fr 1fr 1fr 1fr 1fr;
    grid-template-areas:
      "hd hd hd hd hd hd hd hd hd"
      "nav main main main main main main main main"
      ". ft ft ft ft ft ft ft ft";
}

S piko . povemo, da bo tisti prostor ostal prazen.
